MagicLLM Privacy Policy

Effective date: May 9, 2026

MagicLLM is designed as a local-first AI assistant. The app does not operate, upload to, or store user data on a MagicLLM private server. Chat history, selected model references, local settings, captured images, downloaded model files, and other app data are stored on the user's device unless the user chooses a feature that sends data to a third-party or user-configured service.

1. Scope

This Privacy Policy explains how MagicLLM handles information on Android phones and Wear OS companion devices. It covers the app's local AI model features, remote endpoint option, voice input, image input, file/model management, media/search tools, WhatsApp notification reader, device actions, and Wear OS synchronization.

2. Local-First Data Handling

MagicLLM does not send user data to a MagicLLM-owned private server. By default, local model inference, local chat state, local model configuration, downloaded or imported models, tokenizer files, temporary camera images, and app preferences remain on the user's device.

Some optional features require internet access or communication with services selected by the user. When those features are used, the relevant request data is sent directly from the user's device to the chosen service, not to a MagicLLM private server.

3. Information Stored On Device

MagicLLM may store the following information locally on the user's device:

  • Chat messages, assistant responses, tool results, and conversation state.
  • User-selected model file URIs and locally copied model/tokenizer files.
  • Remote endpoint settings entered by the user, such as an OpenAI-compatible base URL.
  • API tokens or account tokens entered by the user for optional services, such as gated model downloads or user-selected remote endpoints.
  • Temporary camera images, image attachments, media cache files, and generated local file references.
  • Recent WhatsApp notification summaries captured only after the user enables Android Notification Listener access.
  • App settings, feature toggles, backend selection, and connection status.

4. Information Sent Outside The Device

MagicLLM does not upload user data to a MagicLLM private backend. Data may leave the device only when a user uses a feature that necessarily contacts another service. Examples include:

  • User-configured remote inference: if the user selects a remote OpenAI-compatible backend, prompts, conversation context, selected image/text inputs, and related request data are sent directly to the configured endpoint, such as a local Ollama server, llama.cpp server, OpenAI-compatible service, or another endpoint entered by the user.
  • Web, media, and model search tools: search terms and request metadata may be sent to public search/media/model services when the user invokes those tools.
  • Model and tokenizer downloads: URLs, repository identifiers, and download requests may be sent to Hugging Face or other download hosts selected by the user.
  • Wear OS sync: selected chat, prompt, device, or media data may be sent through Google Play Services' Wear OS data layer to the user's paired watch.
  • Phone, email, WhatsApp, maps, browser, and media intents: when the user asks the app to open another app or service, the relevant content may be passed to that app for the requested action.

Third-party services process data under their own privacy policies. Users should only configure remote endpoints and third-party services they trust.

Vulkan hardware features Requires compatible Vulkan compute support for local native model acceleration. This is a device capability requirement, not personal data collection.

6. Optional Device Actions

MagicLLM includes tools that can open Android system screens or other apps for user-requested tasks such as dialing a number, composing email, opening WhatsApp with a prefilled message, launching maps, changing brightness or volume, controlling media, setting alarms/timers, opening web pages, copying text to the clipboard, or downloading media/model files.

These actions are initiated in response to user requests. When another app or Android system component is opened, that app or component may receive the content needed to complete the action.

7. Remote AI Endpoint Mode

MagicLLM can run local models on the device. It can also connect to a user-configured OpenAI-compatible endpoint. Remote mode is optional. If remote mode is enabled, the user's prompts, relevant conversation context, selected attachments, and endpoint credentials may be sent directly to the configured endpoint.

MagicLLM does not control third-party or user-hosted endpoints. Users are responsible for choosing endpoints they trust and for understanding those endpoints' privacy practices.

8. Local Models, Files, And Downloads

MagicLLM supports local model loading and model/tokenizer downloads. Large model files, tokenizer files, projector files, and imported documents remain on the user's device unless the user shares them, sends them to a selected remote endpoint, or downloads them from a third-party host.

Downloads requested through Android's DownloadManager are handled by Android and the selected download host. MagicLLM does not mirror downloaded files through a MagicLLM private server.

9. Chat History And Generated Content

Chat messages, assistant responses, generated text, tool outputs, and settings are stored locally on the user's device. The user can remove app data by clearing MagicLLM's app storage or uninstalling the app. If Android backup is enabled for the device, some app data may be included in Android backup according to Android's backup settings and the app's backup configuration.

10. Children

MagicLLM is not intended to collect personal information from children. The app does not knowingly collect or upload children's data to a MagicLLM private server.

11. Security

MagicLLM stores data locally using Android app storage and Android platform security controls. Some sensitive settings may use Android security libraries when configured by the app. Users should protect their device lock screen, only install the app from trusted sources, and only configure remote endpoints and tokens they trust.

12. User Controls

  • Users can deny or revoke Android runtime permissions in system settings.
  • Users can disable notification listener access in Android notification access settings.
  • Users can stop foreground voice services from the app or Android notification/system controls.
  • Users can remove local app data by clearing storage or uninstalling the app.
  • Users can avoid remote data transfer by using local model mode and not invoking internet-backed tools.
  • Users can remove or change remote endpoint URLs and tokens in app settings.

13. No Sale Of Personal Data

MagicLLM does not sell user personal data. MagicLLM does not operate an advertising profile service, analytics broker, or private cloud database for user content.

14. Changes To This Policy

This policy may be updated when the app's features, permissions, or data handling change. The effective date at the top of this document identifies the current version.

15. Contact

For privacy questions, contact the developer or distributor of MagicLLM through the same channel where the app was provided.
